第一章 概论

1.1 数据结构概述

数据结构的概念

  1. 基础概念
  • 数据：计算机可识别、存储和加工的信息载体 文字、表格、图像等
    *数据的组成（从大到小）
  • 数据项：具有独立意义的不可分割最小单位 学生记录中的“姓名”字段
  • 数据元素：数据的基本单位，作为整体处理 如学生记录
  • 数据对象：相同类型数据元素的集合。
  1. 数据结构组成：
  • 数据逻辑结构：从逻辑关系描述数据
  • 数据存储结构：逻辑结构在储存（映像）。
  • 数据运算：对数据施加的操作（检索、插入、删除等）。

数据结构的组成

1.数据逻辑结构

  1. 常见的逻辑

    • 集合：元素同属一个集合，无其他关系。
    • 线性结构：元素间一对一关系 如学生成绩表
    • 树形结构：元素间一对多关系 如例1.2的层次结构
    • 图形结构：元素间多对多关系 如例1.3的网状结构

  2. 表示方法 二元组表示法：

    • $S=(D,R)$
    • $D$：数据
    • $R$：元素间相邻关系（`` 有向，`(a,b)` 无向）
2.存储结构
  1. 常见的储存结构
存储结构 核心思想 优点 缺点
顺序存储结构 连续存储单元存放元素一整块内存
逻辑相邻=物理相邻		 节省空间、支持随机存取 插入/删除需移动元素，不便修改
链式存储结构 结点单独存储，附加指针域记录相邻结点地址 插入/删除仅修改指针，无需移动元素 空间利用率低
不支持随机存取
索引存储结构 主数据表+索引表（关键字→地址），索引表有序可二分查找 查找效率高 需额外空间建立索引表
哈希(散列)存储 用哈希函数 $h(key)$ 计算元素存储地址，不存储元素间逻辑关系 查找速度快 需处理哈希冲突
依赖哈希函数设计

3. 数据运算的组成
  • 运算定义：抽象描述运算功能（基于逻辑结构）。
  • 运算实现：具体算法（基于存储结构）。
  • 思想：运算定义与实现分离，便于软件开发。

常见的数据类型

  1. 数据类型（C/C++常用类型）
    高级语言中规定数据取值范围、存储方式和运算的概念：

    • 基本数据类型：int、float、double、char 等。
    • 指针类型：直接操作变量地址，用于动态空间分配（malloc/free）。
    • 数组类型：同类型数据的有序序列，支持下标访问。
    • 结构体类型：由多个数据项组成，用于描述复杂对象（如学生、教师）。
    • 共用体类型：多个成员共享同一段内存空间。
    • 自定义类型：typedef 为已有类型取别名，简化代码。

  2. 抽象数据类型（ADT）

  • 定义：数学模型名 + 在模型下的一组操作
  • 组成：数据对象 + 运算定义

1.2 算法和算法分析

算法及其描述

1. 算法的基本概念
  • 算法的定义：
    算法（Algorithm）是对特定问题求解步骤的一种描述，它是指令的有限序列。
  • 算法的五个重要特性
    1. 有穷性：算法必须在执行有限步后结束，每一步都在有限时间内完成。
    2. 确定性：每一条指令必须有确切含义，无二义性。
    3. 可行性：算法中的操作可以通过已实现的基本运算执行有限次来实现。
    4. 输入性：有零个或多个输入。
    5. 输出性：有一个或多个输出（与输入有特定关系的量）。
  • 算法设计目标
    1. 正确性：满足预定功能需求。
    2. 可读性：易于人理解，方便交流和维护。
    3. 健壮性：对非法输入能做出反应和处理，不产生预料外的结果。
    4. 高效性：执行时间短，存储空间需求低。
2. 时间复杂度分析 (Time Complexity)
  • 定义 大 O 表示法
    用于描述算法执行时间随问题规模 $n$ 增长的变化趋势。
    若存在 $c$ 和 $n_0$，使得当 $n \ge n_0$ 时，$T(n) \le c \cdot f(n)$，则记作 $T(n) = O(f(n))$。
  • 常见复杂度量级排序
${% raw %}$O(1) < O(\log_2 n) < O(n) < O(n \log_2 n) < O(n^2) < O(n^3) < O(2^n) < O(n!)${% endraw %}$
  • 分析规则
    • 加法规则：$T(n) = T_1(n) + T_2(n) = O(\max(f(n), g(n)))$（用于并列结构）。
    • 乘法规则：$T(n) = T_1(n) \times T_2(n) = O(f(n) \times g(n))$（用于嵌套循环）。

[!NOTE] 分析技巧
算法的时间复杂度通常由其最深层循环内的基本运算语句决定。

3. 空间复杂度分析 (Space Complexity)

  • 定义
    算法在运行过程中临时占用的存储空间大小。记作：$S(n) = O(g(n))$。

  • 计算规则

    • 形参不计入：形参占用的空间在调用者处分配，不计入该算法的空间复杂度。
    • 原地工作 (In-place)：如果算法所需的辅助空间相对于输入规模 $n$ 是常数（即 $O(1)$），则称该算法为原地工作算法。

1.3 数据结构程序设计

1. 数据结构程序设计步骤

一个完整的程序开发过程通常分为三个核心阶段，体现了从“抽象”到“具体”的落地：

  1. 分析逻辑结构
    • 明确问题中数据元素的关系（如线性、集合、树形等）。
    • 采用 抽象数据类型 (ADT) 描述，定义需要哪些基本运算。
  2. 设计存储结构
    • 根据逻辑结构和运算需求，选择物理实现方式（如顺序存储、链式存储）。
    • 在 C/C++ 中表现为定义具体的结构体（struct 或 typedef struct）。
  3. 设计实现算法
    • 在选定的存储结构上，用编程语言编写函数来实现 ADT 定义的算法。
2.应用程序的层级结构

教材通过 图 1.23 展示了良好的程序封装思想：

  • 底层：存储结构 —— 数据的物理定义。
  • 中层：基本运算函数 —— 直接对存储结构进行操作（如增删改查）。对上层隐藏内部细节。
  • 上层：应用程序/主函数 —— 负责具体的业务逻辑。它不直接碰底层数据，而是通过调用“基本运算函数”来完成任务。

第二章 线性表

2.1 线性表的基本概念

线性表的定义

线性表是由 $n$ ($n \ge 0$) 个相同类型数据元素组成的有限序列。

  • 空表：当 $n=0$ 时，称为空表，记为 () 或 Φ。
  • 逻辑表示：$(a_1, a_2, \dots, a_i, \dots, a_n)$。
  • 元素关系：
    • $a_1$ 是**开始元素**，无前驱。
    • $a_n$ 是**终端元素**（尾元素），无后继。
    • 对于任意相邻元素 $$，称 $a_i$ 是 $a_{i+1}$ 的前驱元素，$a_{i+1}$ 是 $a_i$ 的后继元素。
线性表的逻辑特征
  1. 若不为空，则有且仅有一个开始元素和终端元素。
  2. 除开始元素外，每个元素有且仅有一个直接前驱。
  3. 除终端元素外，每个元素有且仅有一个直接后继。
  4. 每个元素在表中有唯一的序号（下标）。
线性表的基本运算

通常记线性表为 List 或 L，基本运算包括：

运算符 函数定义 功能描述
初始化 InitList(&L) 建立一个空表 L。
销毁 DestroyList(&L) 释放线性表 L 占用的内存空间。
求长度 GetLength(L) 返回线性表 L 的长度。
求元素 GetElem(L, i, &e) 返回线性表 L 中第 i 个元素的值。
按值查找 Locate(L, x) 查找第一个值为 x 的元素序号，找不到返回 0。
插入 InsElem(&L, x, i) 在第 i 个位置插入新元素 x ($1 \le i \le n+1$)。
删除 DelElem(&L, i) 删除第 i 个元素 ($1 \le i \le n$)。
输出 DispList(L) 按先后次序输出所有元素值。

  1. 寻找线性表的一个元素
  • 功能： 获取顺序表中第  i  个逻辑位序的元素，无效位序时返回特殊值。
  • 时间复杂度：O(1)
  • 说明：
    • 逻辑位序  i  范围为  [1, L.length] ，数组下标为  i-1 。
    • 有效时用引用型  e  带回元素值并返回  1 （真），无效时返回  0 （假）。
  1. 按值查找运算
  • 功能： 在顺序表  L  中查找第一个值为  x  的元素，返回其逻辑位序；未找到则返回  0 。
  • 物理下标与逻辑序号的关系： 数组下标从 0 开始，逻辑序号从 1 开始，需做  i+1  转换
  • 时间复杂度： O(n)（最坏情况需遍历全表）
  1. **插入元素
  • 功能 ： 将新元素  x  插入到顺序表  L  中逻辑序号为  i  的位置：
  • 实现逻辑：
    • 若  i  无效（范围错误或表已满），返回 0（插入失败）。
    • 若  i  有效，将  L.data[i-1] ~ L.data[L.length-1] 均后移一位，在  L.data[i-1]  处插入  x ，表长度 +1，返回 1（插入成功）。
  • 参数合法性：  i  必须满足 $1 \le i \le L.length+1$ （可插入到表尾之后），且表未满（$L.length < MaxSize$）
  • 时间复杂度：O(n)
